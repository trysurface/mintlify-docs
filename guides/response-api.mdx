The Response API allows you to programmatically retrieve responses from your forms. You can fetch all responses for an entire environment or filter them by a specific form.

## Step 1: Get the API key

To use the Response API, you'll first need to generate an API key from the Surface Platform.

<Frame>
  <img className="block" src="/images/response-api/1.png" alt="Get API key from Surface" />
</Frame>

1. Log in to your Surface account.
2. Go to **Settings** > **API Keys**.
3. Click on **+ Add API Key**.
4. Give your key a name and click **Create**.
5. **Copy and note down the API key** immediately, as it won't be shown again.

## Step 2: Send requests to the Response API

Once you have your API key, you can use it to authenticate your requests.

### Authentication

All requests to the Response API require an API key passed in the `x-api-key` header.

<CodeGroup>
```bash Header
x-api-key: your-api-key-here
```
</CodeGroup>

## Fetching Responses

You can fetch responses at two levels: environment-wide or for a specific form.

### Fetch all responses for an environment

To get all responses across all forms in your current environment, make a GET request to the `/responses` endpoint without any parameters.

<CodeGroup>
```bash cURL
curl -X GET 'https://api.withsurface.com/api/v1/responses' \
  -H 'x-api-key: your-api-key-here'
```
</CodeGroup>

### Fetch responses for a specific form

To filter responses for a single form, use the `formId` query parameter.

<CodeGroup>
```bash cURL
curl -X GET 'https://api.withsurface.com/api/v1/responses?formId=your-form-id' \
  -H 'x-api-key: your-api-key-here'
```
</CodeGroup>

<Note>
  When fetching by `formId`, the API returns a maximum of 10 responses per request.
</Note>

### Including partial responses

By default, the API only returns completed responses. To include partial or incomplete responses, set the `returnPartials` parameter to `true`.

<CodeGroup>
```bash cURL
curl -X GET 'https://api.withsurface.com/api/v1/responses?formId=your-form-id&returnPartials=true' \
  -H 'x-api-key: your-api-key-here'
```
</CodeGroup>

## Query Parameters

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `formId` | `string` | **Optional.** The ID of the form to fetch responses for. |
| `returnPartials` | `boolean` | **Optional.** Set to `true` to include incomplete responses. Defaults to `false`. |

## Implementation Example (TypeScript)

Here is an example of how to implement a response fetcher using TypeScript and the native `fetch` API.

```typescript
// Example usage with fetch
async function getResponses(params: {
  apiKey: string;
  formId?: string;
  returnPartials?: boolean;
}) {
  const url = new URL("https://api.withsurface.com/api/v1/responses");
  
  if (params.formId) {
    url.searchParams.set("formId", params.formId);
  }
  
  if (params.returnPartials !== undefined) {
    url.searchParams.set("returnPartials", params.returnPartials.toString());
  }
  
  const response = await fetch(url.toString(), {
    method: "GET",
    headers: {
      "x-api-key": params.apiKey,
    },
  });
  
  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }
  
  return await response.json();
}

// Usage: Fetch all responses
const allResponses = await getResponses({
  apiKey: "your-api-key-here"
});

// Usage: Fetch form-specific responses including partials
const formResponses = await getResponses({
  apiKey: "your-api-key-here",
  formId: "your-form-id",
  returnPartials: true
});
```

## Key Points to Remember

- **Authentication**: The `x-api-key` header is mandatory for all requests.
- **Default Behavior**: If no `formId` is provided, the API returns all responses for the environment.
- **Limits**: When filtering by a specific form ID, the response is limited to the 10 most recent entries.
